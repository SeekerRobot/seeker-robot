API reference

      

        REST endpoints
        

          
GET/api/configMCU IP, cam/mic URLs, topic list

          
POST/api/cmd_vel{vx,vy,wz} → /cmd_vel Twist

          
POST/api/stoppublish zero Twist

          
POST/api/tts{text} → /audio_tts_input

          
POST/api/play_wav{path} → /audio_play_file

        

      

      

        WebSocket /ws
        
Server → client frames: status, imu, lidar, log.

        
Client → server frames (same schema as REST): {"type":"cmd_vel",…},
          {"type":"stop"}, {"type":"tts","text":…},
          {"type":"play_wav","path":…}.

      

      

        Direct MCU HTTP endpoints
        
These are served by the ESP32 itself, not this host. URLs come from /api/config:

        

          
GET:80/camMJPEG video

          
GET:81/audio16 kHz 16-bit PCM mic

          
GEThost:8390/lcd_outOLED framebuffer (host → MCU, pulled by MCU)

          
GEThost:8383Speaker PCM (host → MCU, pulled by MCU)
